# MJ Collect — 크롬 웹 크롤링 확장프로그램

CSS 셀렉터로 페이지 안의 항목과 필드를 지정해 데이터를 수집하고, 표 / CSV / JSON / TSV(구글시트 붙여넣기용) / **구글시트 직접 동기화(쓰기/읽기)** 까지 지원하는 크롬 확장입니다.

## 설치 (개발자 모드 로드)

1. 크롬 주소창에 `chrome://extensions` 입력
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램 로드** 클릭
4. 이 폴더(`mjcollect`)를 선택

설치 후 툴바의 **MJ** 아이콘을 클릭하면 별도 창(600×780)이 열립니다. 페이지를 클릭해도 닫히지 않아 크롤링 작업에 편리합니다.

## 사용 방법

### 1) 행 컨테이너 셀렉터
한 항목(상품, 게시글 등)을 감싸는 요소의 CSS 셀렉터를 입력합니다.
- 예: `ul.product-list > li`, `.board .post`, `article.item`

크롬 개발자도구(F12)에서 요소를 우클릭 → **Copy → Copy selector** 로 가져올 수 있습니다(자동 셀렉터는 길거나 깨지기 쉬워, 클래스 기반으로 정리하는 게 안정적입니다).

### 2) 추출할 필드
각 행 내부에서 뽑을 필드들을 정의합니다.

| 칸 | 의미 |
| --- | --- |
| 필드명 | 결과 표의 컬럼 헤더가 됩니다. (예: `제목`, `가격`) |
| 상대 셀렉터 | 행 기준의 CSS 셀렉터. 비워두면 행 자체. (예: `.title a`, `.price`) |
| 추출 방식 | `text` / `html` / `href` / `src` / `value` / 속성명 직접 |

### 3) 크롤링 범위
4가지 모드 중 사이트 형태에 맞는 것을 선택합니다.

#### 현재 탭만
지금 보고 있는 페이지에서 한 번 추출. 가장 안전.

#### URL 패턴
페이지 번호로 URL이 바뀌는 일반적인 사이트.
- 예: `https://example.com/list?page={N}` 시작=1, 끝=10

#### 다음 버튼
URL은 안 바뀌어도 "다음" 버튼이 있는 사이트.
- 셀렉터 예: `a.next`, `button.pagination-next`, `li.next > a`
- **jQuery 사이트 주의**: `.next`(li)만 지정하면 동작 안 할 수 있습니다. `<a>`까지 명시(`li.next > a`).

#### 페이지 번호 클릭 (SPA)
**URL이 안 바뀌는 SPA 페이지네이션** 전용. (jQuery bootpag, React/Vue 페이지네이션 등)
- 셀렉터 템플릿에 페이지 번호 자리를 `{N}`으로 표기
- 시작/끝 페이지 번호 입력 → 각 페이지를 한 칸씩 클릭하며 수집
- 예 (bootpag): `#page-selection li[data-lp="{N}"] a`
- 예 (data-page): `.pagination li[data-page="{N}"] > a`
- 끝 번호가 실제 마지막 페이지보다 크면 자동 중단

### 4) 결과 / 내보내기 / 가져오기
- 팝업 하단에 표 형태로 즉시 표시
- **📊 시트로** — 현재 결과를 구글시트에 바로 append (사전 셋업 필요, 아래 참조)
- **📥 가져오기** — 구글시트의 데이터를 결과 표로 불러오기
- **TSV 복사** — 구글시트 / 엑셀에 그대로 Ctrl+V 가능
- **CSV** — 엑셀 한글 호환 BOM 포함 다운로드
- **JSON** — 파일 다운로드

### 프리셋
사이트별로 자주 쓰는 설정을 이름 붙여 저장 / 불러올 수 있습니다. 모두 브라우저 로컬 저장이며 외부 전송되지 않습니다.

## 📊 구글시트 동기화 셋업 (1회)

확장 프로그램에서 **결과를 구글시트에 직접 추가하거나, 시트의 데이터를 가져올 수 있습니다.** 처음 한 번만 셋업하면 됩니다.

### Step 1. 데이터를 받을 시트 준비
1. 구글시트를 새로 만들거나 기존 시트를 엽니다.
2. 메뉴: **확장 프로그램 → Apps Script** 클릭

### Step 2. Apps Script에 코드 붙여넣기
1. 새로 열린 Apps Script 편집기에서 기본 `Code.gs` 내용을 모두 지웁니다.
2. 이 폴더의 [`apps-script.gs`](./apps-script.gs) 파일 전체 내용을 복사해 붙여넣습니다.
3. 코드 상단의 `TOKEN` 값을 본인만 아는 **긴 랜덤 문자열**로 변경합니다.
   - 예: `mjc_2026_a8x9k2nq3p_change_me`
4. **Ctrl+S** (또는 ⌘+S)로 저장.

### Step 3. 웹앱으로 배포
1. 우상단 **배포 → 새 배포** 클릭.
2. 톱니바퀴(⚙️) → **웹 앱** 선택.
3. 다음 항목 설정:
   - **설명**: `MJ Collect` (자유)
   - **다음 사용자 인증으로 실행**: `본인`
   - **액세스 권한**: `모든 사용자`
4. **배포** 클릭 → 권한 승인 (Google 로그인 → "고급" → "안전하지 않은 페이지로 이동" → 허용).
5. 발급된 **웹 앱 URL**을 복사합니다. (`https://script.google.com/macros/s/.../exec`)

### Step 4. 확장 프로그램에 입력
1. MJ Collect 창에서 **📊 구글시트 연동 (선택)** 카드를 펼칩니다.
2. 입력:
   - **웹 앱 URL**: 위에서 복사한 URL
   - **토큰**: Step 2에서 설정한 것과 **완전히 동일**해야 함
   - **스프레드시트 ID**: 비우면 스크립트가 속한 시트로. 다른 시트에 보내려면 시트 URL의 `/d/[이부분]/edit`에서 ID 추출(시트 URL 통째로 붙여도 자동 추출됨)
   - **시트 탭 이름**: 기본 `MJ Collect`. 없으면 자동 생성됨
3. **연결 테스트** 버튼 → "✓ 연결 OK" 가 나오면 성공
4. 평소엔 크롤링 후 **📊 시트로** 또는 **📥 가져오기** 버튼 사용

### 동작 방식 — 보내기 (📊 시트로)
- 시트 탭이 없으면 **자동 생성**
- 첫 전송 시 **헤더 행** 자동 작성
- 이후 전송은 **마지막 행 다음에 append** (덮어쓰지 않음)

### 동작 방식 — 가져오기 (📥 가져오기)
- 현재 설정된 **시트 탭 전체**를 읽어와 결과 표에 표시
- 첫 행은 **헤더(컬럼명)**, 나머지 행은 데이터로 해석
- 가져온 데이터는 그대로 CSV/JSON 다운로드, TSV 복사, 다시 시트로 보내기 모두 가능
- 빈 시트면 빈 결과 표시 (오류 아님). 시트 탭이 없으면 경고 메시지

### 코드 업데이트 (이미 배포한 분이 가져오기 기능 추가하는 경우)
이미 Apps Script를 배포한 상태에서 가져오기 기능을 쓰려면 새 코드로 업데이트해야 합니다:
1. 시트의 Apps Script 편집기 열기
2. 기존 코드를 모두 지우고 [`apps-script.gs`](./apps-script.gs) 최신 내용 붙여넣기 (TOKEN 값은 그대로 유지)
3. **배포 → 배포 관리 → 편집(연필) → 새 버전 → 배포** (URL이 그대로 유지됨)

### 보안 주의
- 웹 앱 URL과 TOKEN은 외부에 공유하지 마세요. 둘 다 알면 누구나 시트에 쓰거나 읽을 수 있습니다.
- Apps Script 코드를 수정하면 **배포 관리 → 편집(연필) → 새 버전 → 배포**해야 변경사항이 반영됩니다(URL 유지).

### 자주 발생하는 문제
| 증상 | 원인 / 해결 |
|---|---|
| `Invalid token` | 확장의 토큰과 Apps Script의 `TOKEN` 값이 일치하지 않음 |
| `응답 파싱 실패` | 배포 권한이 "모든 사용자"가 아님 (로그인 페이지로 리다이렉트됨) |
| `Spreadsheet not found` | 스크립트가 시트에 바인딩되지 않았는데 시트ID도 안 넣음 |
| `HTTP 401/403` | 배포 후 권한 승인을 안 했거나, URL이 잘못됨 |
| 가져오기 시 `시트 탭이 존재하지 않습니다` | 입력한 "시트 탭 이름"이 실제 시트에 없음. 정확한 탭 이름 입력 |

## SPA 페이지네이션 가이드 (페이지 번호 클릭 모드)

URL이 절대 안 바뀌는 사이트(jQuery bootpag, React Router 내부 상태로 페이지 관리하는 사이트 등)에서 사용합니다.

### 셀렉터 템플릿 찾기
1. F12로 개발자도구 열기
2. 페이지 번호(예: "2") 위에서 우클릭 → 검사
3. 해당 `<a>` 또는 `<li>`를 살펴 **페이지 번호가 들어간 속성**이나 클래스 패턴 확인
4. 그 패턴을 `{N}`으로 치환

### 자주 보는 패턴 모음
| 사이트 패턴 | 셀렉터 템플릿 |
|---|---|
| `<li data-lp="2"><a>2</a></li>` (bootpag) | `li[data-lp="{N}"] a` 또는 `#page-selection li[data-lp="{N}"] a` |
| `<a data-page="2">2</a>` | `a[data-page="{N}"]` |
| `<li class="page-2"><a>...</a></li>` | `li.page-{N} a` |
| `<button data-index="2">2</button>` | `button[data-index="{N}"]` |

### 동작 흐름
- 시작=1, 끝=7 입력 시 → 1번 페이지 클릭(이미 활성이면 무시) → 추출 → 2번 클릭 → 추출 → ... → 7번 추출 → 종료
- 시작=3, 끝=10 입력 시 → 3번 페이지 클릭 → 추출 → 4번 클릭 → ... 처럼 중간부터 가능
- 클릭이 실패해서 더 이상 페이지가 없으면 자동 중단

### 로딩 대기(ms)
- 일반 사이트: `1500` (기본값)
- AJAX가 느린 사이트: `2500~3500`까지 늘림

## 알려진 제약 사항

- **무한 스크롤** 사이트는 미지원 (다음 버튼 또는 페이지 번호 방식이 있는 사이트는 가능)
- 자바스크립트로 늦게 렌더링되는 컨텐츠는 **지연(ms) 값을 늘려** 보세요
- `chrome://`, `chrome-extension://` 등 크롬 내부 페이지에서는 동작하지 않습니다

## 파일 구조

```
mjcollect/
├── manifest.json         # Manifest V3
├── popup.html            # UI
├── popup.css             # 스타일
├── popup.js              # UI / 결과 표시 / 내보내기 / 시트 연동(쓰기·읽기)
├── content.js            # 페이지에 주입되는 추출 / 페이지 클릭 도우미
├── background.js         # 새창 띄우기 + 다중 페이지 자동 순회 (service worker)
├── apps-script.gs        # 구글시트 Apps Script에 붙여넣을 코드 (write/read)
├── icons/                # 16/48/128 아이콘
└── README.md
```
